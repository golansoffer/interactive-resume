# Interactive Resume — Repo Setup Design

**Date:** 2026-05-19
**Scope:** Phase-0 scaffolding only. No 3D libs, no styling system, no domain code, no resume data. The output is an empty, type-strict React + XState + TanStack Router app that builds, type-checks, lints, and tests green.

---

## Goal

Stand up a new repo at `/Users/golan/Documents/repos/interactive-resume/` that:

1. Runs React 19 on Vite with TypeScript at the strictest practical configuration.
2. Routes via `@tanstack/react-router` file-based routing.
3. Has XState v5 wired in (deps installed, no machines yet).
4. Enforces "no TS hacks" as build-failing errors (not warnings) at three layers: `tsc`, `oxlint`, and a suppressor-scan script.
5. Carries forward the Iron Laws and supporting rules from `real-time-players-app/CLAUDE.md`, adapted for a single-package client-only app.
6. Carries forward the relevant agent definitions from `real-time-players-app/.claude/agents/`.

What is explicitly **out of scope** for this setup phase:

- 3D / graphics libraries (three.js, react-three-fiber, drei, GSAP, Pixi, etc.) — separate session.
- Styling system (Tailwind, CSS modules, vanilla-extract, etc.) — separate session.
- XState machine for company-scene navigation — needs domain design first.
- Resume data shape (companies, roles, fields) — needs design.
- Server, API, persistence, auth — none planned.
- CI workflow files, pre-commit hooks (Husky / simple-git-hooks).
- Architecture docs (`docs/architecture.md`, `docs/primitives.md`) — referenced in CLAUDE.md but written later.

```json
    "strict": true,
    "noUncheckedIndexedAccess": true,

    "noPropertyAccessFromIndexSignature": true,
    "noFallthroughCasesInSwitch": true,

    "noImplicitReturns": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "verbatimModuleSyntax": true,
    "isolatedModules": true,
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "skipLibCheck": true,
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true
  },
  "exclude": ["node_modules", "dist", "build"]

```

`tsconfig.json` (app):

```json
{
  "extends": "./tsconfig.base.json",
  "compilerOptions": {
    "noEmit": true,
    "jsx": "react-jsx",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "types": ["vite/client"],
    "baseUrl": ".",
    "paths": { "@/*": ["./src/*"] }
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

`noEmit: true` lives on the app config (not the base) because `tsconfig.node.json` uses `composite: true`, which is incompatible with `noEmit`. `tsc -b` type-checks both projects without producing JS output for the app, and only writes `.tsbuildinfo` for the node config.

`tsconfig.node.json` (Vite config compilation):

```json
{
  "extends": "./tsconfig.base.json",
  "compilerOptions": {
    "composite": true,
    "lib": ["ES2022"],
    "types": ["node"]
  },
  "include": ["vite.config.ts"]
}
```

## Lint configuration

`.oxlintrc.json` — every category that catches a class of TS abuse is `error`, not `warn`. Per user directive: "errors, not warnings".

```json
{
  "$schema": "./node_modules/oxlint/configuration_schema.json",
  "ignorePatterns": [
    "src/routeTree.gen.ts",
    "**/dist/**",
    "**/build/**",
    "**/node_modules/**"
  ],
  "plugins": ["typescript", "react", "import", "unicorn", "oxc"],
  "categories": {
    "correctness": "error",
    "suspicious": "error",
    "pedantic": "error"
  },
  "rules": {
    "typescript/ban-ts-comment": "error",
    "typescript/consistent-type-imports": "error",
    "typescript/no-explicit-any": "error",
    "typescript/no-non-null-assertion": "error",
    "typescript/no-non-null-asserted-nullish-coalescing": "error",
    "typescript/no-unsafe-argument": "error",
    "typescript/no-unsafe-assignment": "error",
    "typescript/no-unsafe-call": "error",
    "typescript/no-unsafe-member-access": "error",
    "typescript/no-unsafe-return": "error",
    "no-console": "warn",
    "react/react-in-jsx-scope": "off"
  },
  "overrides": [
    {
      "files": ["**/services/**/*.ts", "**/services/**/*.tsx", "scripts/**/*.mjs"],
      "rules": { "no-console": "off" }
    }
  ],
  "settings": { "react": { "version": "detect" } },
  "env": { "browser": true, "es2024": true }
}
```

`scripts/check-suppressors.mjs` — a tree scan that exits non-zero if any of these appear anywhere under `src/` (and root config files):

- `@ts-ignore`, `@ts-expect-error`, `@ts-nocheck`
- `eslint-disable`, `oxlint-disable` (line, block, file scope)
- `as any`, `as unknown as`
- `as NonNullable<`
- Postfix `!` access patterns (best-effort regex: `\w![.[]`, `!\s*\.`, `!\s*\?\.`, `!\s*\[`, `!\s*\)`)

Catches what `oxlint` may miss (cast chains, definite-assignment usage in code outside lint rules).

`oxlint` rule `typescript/no-non-null-assertion` already covers the postfix `!` for direct usages; the script provides a second defense and catches `as any` chains.

## Scripts

`package.json` scripts:

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsc -b && vite build",
    "preview": "vite preview",
    "typecheck": "tsc -b --noEmit",
    "lint": "oxlint",
    "lint:suppressors": "node scripts/check-suppressors.mjs",
    "test": "vitest run",
    "test:watch": "vitest",
    "check": "pnpm typecheck && pnpm lint && pnpm lint:suppressors && pnpm test"
  }
}
```

`pnpm check` is the single gate. Green here means everything is green.

## Routing

File-based via `@tanstack/router-plugin/vite`. The plugin watches `src/routes/` and regenerates `src/routeTree.gen.ts` on change. The generated file is gitignored.

Initial routes:

- `src/routes/__root.tsx` — root layout with `<Outlet />`. Mounts `<TanStackRouterDevtools />` in dev.
- `src/routes/index.tsx` — placeholder landing page rendering the text `"interactive resume — scaffolding"`. No styling yet.

`src/main.tsx` constructs the router from `routeTree.gen.ts` and renders into `#root`.

`vite.config.ts` wires:

- `@vitejs/plugin-react`
- `@tanstack/router-plugin/vite` with `target: 'react'` and `autoCodeSplitting: true`
- Path alias `@` → `src` (mirroring tsconfig `paths`)
